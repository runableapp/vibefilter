English

1. Introduction

This Privacy Policy describes how VibeFilter (also known as "걸러드림"), a Chrome browser extension developed by Runable.app, handles user data. We are committed to protecting your privacy and being transparent about our data practices.

2. Extension Overview

VibeFilter is a Chrome extension that aggregates popular posts from Korean online communities and recommends articles based on the user's reading preferences. The extension runs entirely within the browser — there is no external server. All scraping, machine learning, and recommendation happen inside the Chrome extension itself.

3. Data Collection

3.1 Data We Collect

VibeFilter processes the following types of data, all of which remain inside your browser:

  • Browsing interaction data: Which articles you click, skip, or mark as "not interested" within the extension's side panel.
  • User preferences: Extension settings such as display options and blocked authors, stored via Chrome's storage API.
  • Collected article metadata: Titles, authors, URLs, and community source of aggregated posts, stored in the browser's IndexedDB.
  • Machine learning model data: Locally trained model parameters (TF-IDF vectors, logistic regression weights) derived from your reading patterns, stored in IndexedDB.

3.2 Data We Do NOT Collect

  • We do not collect any personally identifiable information (PII).
  • We do not collect browsing history outside of the extension's side panel.
  • We do not collect passwords, financial information, or authentication tokens.
  • We do not collect form data or user-generated content.
  • We do not use cookies or tracking technologies.

4. Data Usage

All data processed by VibeFilter is used solely for the following purposes:

  • Article recommendation: Your click and interaction patterns are used to train a local machine learning model that predicts which articles you may find interesting.
  • Read tracking: Previously read articles are tracked so they can be hidden or de-emphasized.
  • Preference reports: Generating local reports showing your preferred keywords, communities, and authors.

5. Data Storage & Processing

  • 100% In-Browser: All data is stored and processed entirely within Chrome. The extension does not communicate with any external server or backend process.
  • No external servers: No user data is transmitted to any external server, cloud service, or third party.
  • No remote code execution: The extension does not load or execute any remotely hosted code.
  • Browser storage: Article data and learning data are stored in the browser's IndexedDB. Settings are stored via Chrome's storage API.

6. Data Sharing

We do not share, sell, transfer, or disclose any user data to any third party, under any circumstances. Since all data stays inside your browser, there is no data to share.

7. Permissions Justification

VibeFilter requests the following Chrome permissions, each necessary for core functionality:

  • sidePanel – Required to display the article list in Chrome's side panel interface.
  • activeTab – Required to open article links in the current browser tab when clicked.
  • tabs – Required to manage tab navigation when opening articles.
  • storage – Required to save user preferences and extension settings locally.
  • alarms – Required to schedule periodic background scraping of community sites.
  • offscreen – Required to create an offscreen document for HTML parsing during scraping.
  • host_permissions – Required to access public bulletin board listing pages of supported Korean community sites. Only the specific community site domains listed in the manifest are accessed.

8. Network Communication

The extension's only network communication is fetching public bulletin board listing pages from the supported community sites. It does not access pages requiring login and does not use any user account information. No data is sent to any server.

9. User Control & Data Deletion

  • You can delete all stored data at any time from the "Data Management" section in the extension's settings page.
  • You can reset extension preferences through the settings page.
  • Uninstalling the extension automatically removes all stored data (IndexedDB and Chrome storage).

10. Children's Privacy

VibeFilter is not directed at children under the age of 13. We do not knowingly collect data from children.

11. Changes to This Policy

We may update this Privacy Policy from time to time. Any changes will be reflected on this page with an updated "Last Updated" date. Continued use of the extension after changes constitutes acceptance of the revised policy.

12. Contact

If you have any questions or concerns about this Privacy Policy, please contact us:

  • Developer: Runable.app
  • Website: https://runable.app
  • Report Issues: report-issues.html
한국어

1. 소개

본 개인정보 처리방침은 Runable.app이 개발한 크롬 브라우저 확장 프로그램 걸러드림(VibeFilter)의 사용자 데이터 처리 방식을 설명합니다. 당사는 사용자의 개인정보를 보호하고 데이터 처리 방식에 대해 투명하게 안내하기 위해 최선을 다합니다.

2. 확장 프로그램 개요

걸러드림(VibeFilter)은 한국 온라인 커뮤니티의 인기 게시글을 수집하고, 사용자의 읽기 패턴을 학습하여 관심 있는 글을 추천하는 크롬 확장 프로그램입니다. 모든 기능이 브라우저 안에서 실행되며, 외부 서버를 사용하지 않습니다. 게시글 수집, 머신러닝, 추천 기능 모두 크롬 확장 프로그램 내부에서 처리됩니다.

3. 데이터 수집

3.1 수집하는 데이터

걸러드림(VibeFilter)은 다음과 같은 데이터를 처리하며, 모든 데이터는 사용자의 브라우저 안에만 저장됩니다:

  • 사용 패턴 데이터: 확장 프로그램 사이드 패널에서 클릭, 무시, "관심없음" 표시한 게시글 정보.
  • 사용자 설정: 표시 옵션, 차단 작성자 등 확장 프로그램 설정(Chrome storage API 사용).
  • 수집된 게시글 메타데이터: 제목, 작성자, URL, 출처 등 수집된 게시글 정보(브라우저 IndexedDB 저장).
  • 머신러닝 모델 데이터: 사용자의 읽기 패턴으로 학습된 모델 파라미터(TF-IDF 벡터, 로지스틱 회귀 가중치, IndexedDB 저장).

3.2 수집하지 않는 데이터

  • 개인 식별 정보(PII)를 수집하지 않습니다.
  • 확장 프로그램 사이드 패널 외부의 브라우징 기록을 수집하지 않습니다.
  • 비밀번호, 금융 정보, 인증 토큰을 수집하지 않습니다.
  • 양식 데이터나 사용자 생성 콘텐츠를 수집하지 않습니다.
  • 쿠키나 추적 기술을 사용하지 않습니다.

4. 데이터 이용 목적

걸러드림(VibeFilter)이 처리하는 모든 데이터는 오직 다음 목적으로만 사용됩니다:

  • 게시글 추천: 클릭 및 상호작용 패턴을 사용하여 관심 있는 게시글을 예측하는 로컬 머신러닝 모델을 학습합니다.
  • 읽은 글 추적: 이미 읽은 게시글을 숨기거나 흐리게 표시하기 위해 추적합니다.
  • 선호도 리포트: 선호 키워드, 커뮤니티, 작성자 등의 분석 리포트를 로컬에서 생성합니다.

5. 데이터 저장 및 처리

  • 100% 브라우저 내 처리: 모든 데이터는 크롬 브라우저 안에서만 저장되고 처리됩니다. 확장 프로그램은 외부 서버나 백엔드 프로세스와 통신하지 않습니다.
  • 외부 서버 미사용: 어떤 사용자 데이터도 외부 서버, 클라우드 서비스, 제3자에게 전송되지 않습니다.
  • 원격 코드 실행 없음: 확장 프로그램은 원격 호스팅된 코드를 로드하거나 실행하지 않습니다.
  • 브라우저 저장소: 게시글 데이터와 학습 데이터는 브라우저의 IndexedDB에 저장됩니다. 설정은 Chrome storage API를 통해 저장됩니다.

6. 데이터 공유

당사는 어떠한 상황에서도 사용자 데이터를 제3자와 공유, 판매, 이전 또는 공개하지 않습니다. 모든 데이터는 사용자의 브라우저 안에만 존재하므로 공유할 데이터가 없습니다.

7. 권한 사용 근거

걸러드림(VibeFilter)은 핵심 기능을 위해 다음 Chrome 권한을 요청하며, 각 권한의 필요성은 다음과 같습니다:

  • sidePanel – 크롬 사이드 패널에 게시글 목록을 표시하기 위해 필요합니다.
  • activeTab – 클릭 시 현재 탭에서 게시글 링크를 열기 위해 필요합니다.
  • tabs – 게시글 열기 시 탭 탐색을 관리하기 위해 필요합니다.
  • storage – 사용자 설정과 확장 프로그램 환경설정을 로컬에 저장하기 위해 필요합니다.
  • alarms – 커뮤니티 사이트의 주기적 백그라운드 스크래핑을 예약하기 위해 필요합니다.
  • offscreen – 스크래핑 중 HTML 파싱을 위한 오프스크린 문서를 생성하기 위해 필요합니다.
  • host_permissions – 지원되는 한국 커뮤니티 사이트의 공개 게시판 목록 페이지에 접근하기 위해 필요합니다. 매니페스트에 명시된 커뮤니티 사이트 도메인만 접근합니다.

8. 네트워크 통신

확장 프로그램의 유일한 네트워크 통신은 지원되는 커뮤니티 사이트의 공개 게시판 목록 페이지를 가져오는 것입니다. 로그인이 필요한 페이지에는 접근하지 않으며, 사용자 계정 정보를 사용하지 않습니다. 어떤 데이터도 서버로 전송되지 않습니다.

9. 사용자 통제 및 데이터 삭제

  • 확장 프로그램 설정 페이지의 "데이터 관리" 섹션에서 언제든 모든 저장된 데이터를 삭제할 수 있습니다.
  • 설정 페이지에서 확장 프로그램 환경설정을 초기화할 수 있습니다.
  • 확장 프로그램을 제거하면 모든 저장 데이터(IndexedDB, Chrome storage)가 자동으로 삭제됩니다.

10. 아동 개인정보

걸러드림(VibeFilter)은 만 13세 미만의 아동을 대상으로 하지 않습니다. 당사는 아동의 데이터를 의도적으로 수집하지 않습니다.

11. 개인정보 처리방침 변경

당사는 본 개인정보 처리방침을 수시로 업데이트할 수 있습니다. 변경 사항은 이 페이지에 "최종 업데이트" 날짜와 함께 반영됩니다. 변경 후 확장 프로그램을 계속 사용하면 수정된 방침에 동의한 것으로 간주됩니다.
